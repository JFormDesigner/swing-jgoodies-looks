:: Building ::

In some cases, there's a need to extend or modify the original classes. This document explains how to build the binary code from the library sources.

The Looks package ships with precompiled JARs for the following l&f sets:

  • all JGoodies l&s
  • the JGoodies Plastic l&f family
  • the JGoodies Windows l&f.
You can use these JARs if the original classes are fine for you. Or you can extract the contained classes and integrate them with your application JAR.

Compiling the Sources for Java 1.4 or Later

If you want to use the binary Looks with Java 1.4 or later, just compile all sources with an 1.4 compiler against the J2SE 1.4 runtime libraries.

Compiling the Sources for Java 1.3 or Later

If you want to use the binary Looks with Java 1.3 or later, you must compile some sources against the J2SE 1.3 runtime libraries and others against the 1.4 runtime.

Due to a bug in Sun's javac 1.3 some classes will be rejected by this compiler. I therefore recommend to use an 1.4 compatible compiler for both the 1.3 and the 1.4 classes; just switch the boot classpath. You can find a default build process in the accompanying ANT build script.

The following classes must be compiled against the Java 1.4 runtime:

  • com.jgoodies.plaf.ExtBasicArrowButtonHandler
  • com.jgoodies.plaf.plastic.PlasticSpinnerUI
  • com.jgoodies.plaf.windows.ExtWindowsSpinnerUI

Shrink & Obfuscate

If you want to shrink or obfuscate your deployed code you may need to exclude the 1.4 classes from the shrink/obfuscation process.

IDE Integration

I recommend to refer the looks-x.x.x.JAR file in your IDE; this will work fine in both 1.3 and 1.4 environments. If you are working with 1.4 only, you can use the Looks sources too.
