JGoodies Looks, Version 1.1.2

The JGoodies look&feels make your Swing applications and applets look better. They have been optimized for readability, precise micro-design and usability.

Requirements

The JGoodies Looks requires Java 1.3 or later.

Getting Started

First, download and run the JGoodies Looks Demo. Then read the user's guide, and finally check out the tips. If you want to use a JGoodies l&f in NetBeans read here.

Distribution Contents

docs/api   JavaDocs directory
docs/guide   User's Guide
 
examples/lib   library for the simple Looks demo
examples/tiny.jar   precompiled Tiny example
examples/demo.jar   precompiled simple Looks demo
 
src/examples/tiny   sources for the tiny example
src/examples/demo   sources for the simple Looks demo
src/photoshop   PhotoShop files for Plastic icons
 
LICENSE.txt   the BSD license agreement
README.html   this readme file

looks-1.1.2-src.zip   zipped sources for all JGoodies l&fs

looks-win-1.1.2.jar   a JAR containing the JGoodies Windows l&f
plastic-1.1.2.jar   a JAR containing the JGoodies Plastic l&fs

Support

The Looks project is maintained and unofficially supported at JavaDesktop.org, see http://looks.dev.java.net. Customers of the JGoodies Looks Pro and Swing Suite get priority support and can contact us by mail and phone too.

Binary and Source Code

The JGoodies Looks require Java 1.3 or later. The following classes are used for 1.4 or later and cannot be compiled in 1.3 environments:
  • com.jgoodies.plaf.ExtBasicArrowButtonHandler
  • com.jgoodies.plaf.plastic.PlasticSpinnerUI
  • com.jgoodies.plaf.windows.ExtWindowsSpinnerUI
Anyway, I recommend to refer the looks-1.1.2.jar in your IDE. If you want to shrink or obfuscate your deployed code you may need to exclude the 1.4 classes from the shrink/obfuscation process.
